GitHub이란 무엇인가요?

GitHub(깃허브)는 파일을 저장하고 인터넷에 공개할 수 있는 무료 저장소 서비스입니다. 특히 HTML 파일(웹페이지)을 올리면 누구나 인터넷 주소로 접속할 수 있는 웹사이트가 됩니다.

💡 한 줄 요약

GitHub = 파일을 저장하는 구글 드라이브 + 웹사이트로 공개해주는 기능이 합쳐진 서비스

주요 용어 정리

용어쉬운 설명
Repository (레포지토리)내 파일들을 모아두는 폴더. 줄여서 "레포"라고 부릅니다.
Commit (커밋)파일을 저장하는 행위. '저장 + 메모'라고 생각하면 됩니다.
Branch (브랜치)작업 공간. main 브랜치 = 공식 버전입니다.
GitHub Pages레포에 있는 HTML 파일을 웹사이트로 공개해주는 기능입니다.
Actions (액션)파일을 올리면 자동으로 특정 작업을 실행해주는 로봇 기능입니다.
Workflow (워크플로우)Actions가 어떤 순서로 일할지 적어둔 설명서(YAML 파일)입니다.

오늘 한 작업 정리

Claude와 함께 아래 세 가지 작업을 완료했습니다.

① 레포지토리 폴더 구조 설정

class-questions/
  ├── .github/workflows/
      └── deploy.yml  ← 자동 배포 설명서
  ├── articles/  ← HTML 아티클 저장 폴더
  ├── scripts/
      └── generate_index.py  ← 목록 자동 생성
  └── index.html  ← 자동 생성 (직접 수정 ❌)

② GitHub Actions 자동 배포 설정

main 브랜치에 파일을 올리면 자동으로 아래 순서가 실행됩니다.

1
변경 감지
GitHub이 새 파일 업로드를 감지합니다.
2
index.html 자동 생성
generate_index.py가 실행되어 모든 파일을 스캔하고 목록 페이지를 새로 만듭니다.
3
GitHub Pages 배포
완성된 파일이 자동으로 웹에 배포됩니다.
4
접속 가능
약 1~2분 후 웹 주소에서 새 페이지를 확인할 수 있습니다.

자주 실수하는 것들

404 오류가 뜬다 Actions 탭에서 배포가 완료됐는지 확인하세요. 초록 체크(✅)가 뜨면 성공입니다. 파일을 올린 직후에는 1~2분 기다려야 해요.
deploy.yml이 Actions에 안 보인다 파일 위치가 반드시 .github/workflows/ 안에 있어야 합니다. .github/ 바로 아래에 있으면 인식이 안 됩니다.
index.html을 직접 수정했다 index.html은 자동 생성 파일입니다. 다음 배포 때 덮어씌워집니다. 직접 수정하지 마세요.
파일을 올렸는데 목록에 안 보인다 articles/, specs/, reports/ 등 정해진 폴더에 넣어야 자동으로 목록에 추가됩니다. 레포 최상단에 바로 올리면 목록에 나오지 않습니다.
⚠️ 가장 중요한 규칙

index.html은 절대 직접 수정하지 마세요. Actions가 자동으로 덮어씁니다. 아티클 파일만 articles 폴더에 올리면 나머지는 자동으로 처리됩니다.
